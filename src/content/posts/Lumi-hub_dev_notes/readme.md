Lumi-Hub 是一套独立于现有聊天平台的私有化 Agent 交互框架。它不仅是一个聊天前端，而是专为大模型（如基于 AstrBot 的 Agent）打造的本地化多端交互枢纽。

[![GitHub](https://img.shields.io/badge/GitHub-Lumivers%2FLumi--Hub-blue?logo=github)](https://github.com/Lumivers/Lumi-Hub)

---

## 项目背景

传统的第三方 IM 平台（如 QQ、微信、Telegram 等）作为机器人载体时，往往面临平台封控风险、消息拦截以及功能扩展受限等问题。

Lumi-Hub 旨在提供一个纯净、可靠的自建通信链路。Host 端直接作为 AstrBot 的**自定义平台适配器**运行，在本地构建完整的消息收发与任务执行闭环，从而摆脱对外部平台的依赖，实现数据与交互体验的完全自主可控。

## 安装

### 👤 普通用户（推荐）

| 步骤 | 操作 |
|------|------|
| **① 安装 Host 插件** | 在 AstrBot WebUI 插件市场搜索 `lumi-hub-plugin` → 一键安装 |
| **② 启动 AstrBot** | 安装插件后，**必须先启动 AstrBot 本体**，Host 端服务会随之运行 |
| **③ 下载客户端** | 前往 [Releases](https://github.com/Lumivers/Lumi-Hub/releases) 下载最新 `Lumi-Hub-Windows-vX.X.X.zip`，解压后运行 `lumi_client.exe` |
| **④ 自动连接** | 打开客户端后会自动连接后台，端口为8765。**(⚠️ 注意：当前 Beta 阶段客户端仅支持连接本机的 AstrBot，请确保两者在同一台电脑上)** |

### 🛠️ 开发者（本地调试）

```bash
# Windows 环境（以管理员身份运行）
mklink /D "你的AstrBot路径\data\plugins\lumi_hub" "本仓库路径\host"

# Linux / macOS 环境
ln -s "$(pwd)/host" 你的AstrBot路径/data/plugins/lumi_hub
```

> 重启 AstrBot 后插件自动加载。Client 端在 `client/` 目录执行 `flutter run -d windows` 启动调试。

## 架构设计

系统整体采用“轻量前端，重载后端”的设计理念：

- **终端 (Client)**：负责消息呈现、交互输入以及高危操作的授权审批（Human-in-the-loop）。
- **主机 (Host + Agent)**：负责核心运算、模型对接与工具调用。

```text
┌─────────────────── 跨平台终端 ─────────────────────┐
│                                                     │
│   Flutter Client                                    │
│   ├── 对话界面与消息流展示                            │
│   └── 高危操作审批弹窗 (Human-in-the-loop)           │
│                                                     │
└───────────────── WebSocket 通信 ────────────────────┘
                        │
                        ▼
┌─────────────────── 你的服务器 / 主机 ────────────────┐
│                                                    │
│   Host（通信桥梁）                                   │
│   ├── 接收指令并透传至 AstrBot 引擎                   │
│   ├── 将 LLM 响应流式转发至 Client                    │
│   └── 识别工具调用意图，交由本地 MCP Client 处理       │
│                                                    │
│   Agent/MCP（本地执行引擎）                           │
│   ├── 直连各类开源 MCP Server（如查询、文件操作等）     │
│   └── 灾备机制：操作前强制备份至文件同级 `.Lumi_cache`  │
│                                                   │
└───────────────────────────────────────────────────┘
```

## 核心模块职责

### Host — AstrBot 自定义平台适配器（Python）
- 启动 WebSocket Server，承接 Client 连接
- 将消息以平台消息形式提交给 AstrBot（与 QQ 消息进入的流程完全一致）
- 接收 AstrBot 的 LLM 回复，转发回 Client
- 指令分流：判定是否需要调用外部 MCP 工具，高危操作触发审批拦截
- 人格管理：同步 AstrBot 数据库中的人格列表，支持前端直接修改 prompt

### Agent — 执行引擎（原生 MCP 客户端集成）
- **原生接入**：Host 内部集成高性能 MCP Client，直接与各类标准 MCP Server 通信。
- **动态感知**：启动时自动扫描所有已加载 Server 的 Tool 定义，并将其注入 AI 的系统意识中。
- **安全熔断 (Fail-Safe)**：写操作前由于路径确定性逻辑，强制触发同级 `.Lumi_cache` 备份；若备份失败则立即中止操作。
- **热重载支持**：支持在不重启主进程的情况下，通过 WebSocket 接口实时重载 MCP 配置与连接。

### 3. Client 端 (Flutter)
- 支持跨平面运行（现支持 Windows 桌面端，后续规划支持 Android/iOS/macOS）。
- 提供现代化的聊天排版视图，自动适配系统深浅色主题。
- 提供流式输出打字机动效、快捷发送机制。
- **全新升级的 UI 体验**：
  - 接入 `window_manager`，实现冷启动按 0.618 比例居中展示，彻底告别启动白屏闪烁。
  - 现代化卡片式设置面板（iOS/macOS 风格），支持字体缩放调整、快捷设置。
  - 支持类似原生 QQ 的跨气泡长段落自由文本选中（基于 `SelectionArea`）。
  - 精致的悬停交互时间戳，左侧边栏高亮动效，沉浸式的“自动登录中”平滑过渡。
- **生命周期策略灵活可控**：支持在设置面板自定义“点击关闭时最小化到系统托盘”或“直接退出”，同时提供“关闭前端时是否同步关闭 AstrBot 后台进程”的智能开关，兼顾原生体验与极客需求。
  - 默认全量接入 MiSans 无衬线黑体，视觉更加和谐统一。
- **可视化 MCP 配置管理**：
  - 专用「扩展生态 (MCP)」配置页面，采用卡片式布局管理各类服务。
  - 支持直接在 UI 中填入/修改 API Token 等敏感凭证，点击保存即可触发后端热重载。
  - 对接后端 WebSocket API，支持配置的实时获取与下发。

## 核心架构：双轨制执行系统 (Dual-Track System)

Lumi-Hub 采用创新的**双轨制**设计，在“极速响应”与“无限扩展”之间取得了平衡：

### 1. 轻量原生轨 (Native Python Track) —— 肌肉记忆
*   **定位**：针对高频、低危、追求绝对速度的 **本地文件系统 CRUD**（增删改查）。
*   **特性**：直接在 Host 进程内执行，提供 `search_replace` (搜索替换)、分页 `read_file` 等工具。
*   **优势**：0 延迟响应，无复杂的鉴权开销，100% 适配重构任务。

### 2. 重型扩展轨 (原生 MCP Track) —— 标准化生态接口
*   **定位**：针对 **复杂/未知/高危/生态级操作**。
*   **特性**：Host 内部集成 **MCP (Model Context Protocol) Client**，直接挂载各类开源或自建的 MCP Server。
*   **优势**：拒绝黑箱，通信链路完全透明；原生支持人机环路（HitL）审批拦截；无缝接入庞大的 MCP 社区插件生态（MySQL、Docker、GitHub 等）。

## 人格与主题系统

系统采用前后端分离的配置方式：
- **后端人格设定**：由 AstrBot 端维护，决定 AI 的系统设定（System Prompt）、认知模型和可用工具。
- **前端视觉主题**：纯本地化管理，决定聊天界面的视觉表现（头像、气泡颜色、面板风格）。
用户可以将严谨的技术逻辑提示词与个性化的视觉主题自由组合，互不干扰。项目默认提供「流萤」预设主题包用于快速体验。

## 安全机制

开放系统操作权限意味着必须具备严格的安全兜底机制：
1. **自动备份**：任何文件写入、删除或修改动作，在执行前会强制备份到该文件同级目录下的 `.Lumi_cache/` 中。
2. **安全熔断**：系统采用 Fail-Safe 设计，若备份过程因权限或磁盘空间等原因失败，后续破坏性操作将自动中止。
3. **人工审批**：高危操作会在执行前将状态挂起，通过 Client 端弹出具有 Diff 预览功能的玻璃美学审批界面，等待用户手动确认（Human-in-the-loop）。
4. **对话式清理**：用户可通过自然语言指令轻松完成对各目录下缓存数据的释放与管理。
5. **逻辑稳定性**：原生 `search_replace` 工具支持忽略行尾空格差异，极大降低了 AI 因视力误差导致的操作失败率。

## 迭代计划

### 阶段14当前落地（更新于 2026.3.24）

已完成：

1. **历史记录流式分页与首屏落底**：彻底告别历史数据“全量一把梭”，实现平滑向上懒加载。攻克前端渲染与数据下发的时序竞态问题，实现进入聊天页即精准“吸底”看最新消息。
2. **体验护城河保卫战 (AXTree攻坚)**：针对严重干扰注意力的 Windows 版 Flutter 底层 `ui::AXTree` 刷屏报错，在坚决不阉割用户“自由选中并复制文本”核心体验的前提下，通过注入深度绑定的 `ValueKey` 稳住 Element 视图树，完美治愈了重绘狂暴和报错日志。
3. **沉浸式内联附件基建贯通**：剥离杂乱的纯文本附件表述，双端订立规范的 `[类型] 路径|||名称` 结构化通信协议并打通 SQLite 历史落库。前端文件卡片支持原生系统程序唤醒（双击直接弹开电脑上的 PDF/Word），初具生产力级体验。
4. **人格侧边栏减负与顺滑持久化**：移除累赘复用的冗余菜单，升级至带有重力物理反馈的拖拽排序列表，并底层接通 `shared_preferences`，实现个人排版偏好“手指离开即落盘”的永久记忆。
